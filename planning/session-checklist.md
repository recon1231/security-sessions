# Session Checklist

A repeatable checklist to run before, during, and after every live session — keeps quality consistent as you do more of these.

---

## Before the Session

- [ ] Topic finalized and outline written (use [`template.md`](../template.md))
- [ ] Slides/visuals prepared, reviewed for any accidental real/sensitive data
- [ ] Any live demo tested beforehand — never debug live in front of an audience
- [ ] Session promoted at least 2-3 days in advance (X, LinkedIn, Discord, relevant groups)
- [ ] Backup plan if screen-share/tech fails (offline slides, printed notes)
- [ ] Confirm session length and stick to it — better to end 5 minutes early with room for questions than run over

## During the Session

- [ ] Open with the hook, not a long introduction about yourself
- [ ] Check engagement periodically — ask a quick question rather than talking uninterrupted for 30+ minutes
- [ ] Take note of interesting questions as they come up (for the session log afterward)
- [ ] Leave real time for Q&A — don't let prepared content eat all the time

## After the Session

- [ ] Fill out the session log using [`template.md`](../template.md) while it's fresh
- [ ] Note what worked and what to improve next time
- [ ] Identify any follow-up content opportunities (a question that deserves its own writeup/video)
- [ ] Thank attendees / post a short recap on socials if appropriate
- [ ] Add the session to the README index

---

## Common Pitfalls to Avoid
- Overloading a beginner session with jargon — define terms the first time they're used
- Running a live demo against a real system without explicit authorization
- Sharing real screenshots without redacting sensitive details first
- Ending without a clear "what to do next" for the audience (follow, read the writeup, try the tool, etc.)
